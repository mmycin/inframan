# Reporting Issues

Thank you for helping make **InfraMan** better! This guide outlines how to report bugs, suggest features, and ask questions effectively.

## Before Submitting an Issue

Before creating a new issue, please perform the following steps:

1.  **Check Documentation**: Re-read the [README.md](../Readme.md) and other docs in `docs/` to ensure the behavior isn't intended or documented.
2.  **Search Existing Issues**: Search the [GitHub Issue Tracker](https://github.com/mmycin/inframan/issues) to see if the issue has already been reported.
3.  **Update InfraMan**: Ensure you are running the latest version of InfraMan.
4.  **Isolate the Problem**: Try to create a minimal reproducible example.

## How to Report a Bug

When reporting a bug, please use the **Bug Report** template and provide as much detail as possible:

-   **Description**: A clear and concise description of what the bug is.
-   **Steps to Reproduce**: Detailed steps to reproduce the behavior.
-   **Expected Behavior**: What you expected to happen.
-   **Actual Behavior**: What actually happened (include logs or error messages).
-   **Environment**:
    -   OS: (Windows 10/11, Ubuntu 22.04, etc.)
    -   InfraMan Version: (`inframan --version`)
    -   Provider used: (Docker, Podman, etc.)
-   **Screenshots**: If applicable, add screenshots to help explain your problem.

## How to Suggest a Feature

We welcome feature requests! Please use the **Feature Request** template:

-   **Problem Definition**: Is your feature request related to a problem? Please describe.
-   **Proposed Solution**: A clear and concise description of what you want to happen.
-   **Alternatives Considered**: Any alternative solutions or features you've considered.
-   **Additional Context**: Add any other context or screenshots about the feature request here.

## Asking Questions

For general questions about how to use InfraMan, please check the [Discussion](https://github.com/mmycin/inframan/discussions) tab if available, or open an issue using the **Question** template.

---

*Note: Please avoid reporting security vulnerabilities through public issues. Refer to [SECURITY.md](SECURITY.md) for the proper disclosure process.*
